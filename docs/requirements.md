# 万能视频下载网站 — 需求分析文档

> 版本：v1.0 | 更新日期：2026-06-21 | 状态：第一期 MVP 已完成

## 1. 项目背景

许多用户有将视频保存到本地的需求，但各平台存在以下痛点：

- 不支持直接下载
- 下载入口隐藏或操作繁琐
- 不支持批量下载
- 清晰度受限

本项目旨在提供一个**万能视频下载网站**，让用户随时随地（含手机端）快速从各平台下载视频。

## 2. 目标用户

| 用户类型 | 需求 |
|---------|------|
| 普通用户 | 粘贴链接 → 选清晰度 → 下载到本地 |
| 移动端用户 | 手机浏览器直接下载，无需安装 App |
| 学习者/开发者 | 了解 yt-dlp + FastAPI + 现代前端开发模式 |

## 3. 功能需求

## 4. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 解析 < 5s（取决于目标站点） |
| 并发 | 最多 3 个同时下载 |
| 存储 | 无数据库；临时文件 2 小时自动清理 |
| 文件大小 | 单文件默认上限 2GB |
| 安全 | 不存储用户 Cookie；不记录 URL 历史 |
| 合规 | 页面固定展示版权提示 |

## 5. 约束与风险

1. **版权**：工具仅供个人学习研究，用户需尊重视频版权
2. **封号风险**：不支持登录态/Cookie 导入，降低账号关联风险
3. **平台限制**：部分平台可能随时变更策略，下载失败属正常现象
4. **技术选型**：核心依赖 [yt-dlp](https://github.com/yt-dlp/yt-dlp)（170k+ Stars），站在巨人肩膀上

## 6. UI/UX 需求

参考 [ai.codefather.cn/painting](https://ai.codefather.cn/painting) 的设计风格：

- 浅灰背景 + 大量留白
- Hero 大标题 + 蓝色高亮关键词
- 胶囊形 URL 输入框 + 集成蓝色按钮
- 白底圆角卡片 + 轻阴影 + `#标签`
- 主色 `#1677FF`，圆角 12px
- 移动端单列卡片堆叠
- 付费感文案与样式（Pro 入口预留）

## 7. 验收标准（第一期）

- [ ] 输入合法视频 URL 可成功解析并展示信息
- [ ] 可选择不同清晰度/格式
- [ ] 点击下载后显示进度，完成后可保存文件
- [ ] 手机浏览器布局正常
- [ ] 页面包含版权提示
- [ ] 临时文件自动清理机制生效
